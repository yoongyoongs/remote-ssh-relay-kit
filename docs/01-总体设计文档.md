# 总体设计文档

## 1. 目标

本项目的目标是提供一套自建远程 SSH 接入方案，使你可以通过一台公网 Linux 服务器，远程连上另一台没有公网 IP 的 Windows 或 macOS 电脑。

这次重新设计后，Windows 端的要求进一步明确为：

1. 如果系统没有安装 `OpenSSH Server`，程序必须自己安装。
2. 用户点击一键启动后，必须能看到当前正在做什么。
3. 安装过程可以在后台执行，也可以单独弹出一个安装窗口。
4. 无论安装在后台还是单独窗口里跑，主程序都要显示安装信息和最终结果。

## 2. 总体架构

整套系统分为三部分：

1. 操作员端
2. 中转服务器
3. 目标机器

### 2.1 操作员端

操作员端负责：

- 生成自己的 SSH 密钥对
- 部署中转服务器
- 为本次接入准备注册码
- 将配置好的启动包发给目标用户
- 使用最终生成的 SSH 命令连接目标机器

### 2.2 中转服务器

中转服务器负责：

- 提供注册接口
- 为每台目标机器分配唯一远程端口
- 保存设备状态
- 把目标机器的设备公钥写入 `tunnel` 账号的 `authorized_keys`
- 限制每台设备只能监听自己被分配到的端口

### 2.3 目标机器

目标机器负责：

- 准备本机 SSH 服务
- 写入操作员公钥
- 生成设备密钥
- 向中转服务器注册
- 发起反向 SSH 隧道
- 把最终 SSH 命令展示给目标用户

## 3. Windows 端重新设计

## 3.1 旧问题

之前的问题主要在这里：

1. 虽然逻辑上已经包含 OpenSSH 自动安装，但用户感知不够清楚。
2. 主窗口只能看到粗粒度步骤状态，看不到安装过程本身。
3. 缺少“安装在后台跑”与“单独弹出安装窗口”之间的明确控制。

## 3.2 新结构

Windows 端拆成四部分：

1. `start-remote-ssh.bat`
2. `RemoteSshApp.ps1`
3. `RemoteSshWorker.ps1`
4. `InstallOpenSsh.ps1`

### 3.2.1 start-remote-ssh.bat

作用：

- 作为用户双击的唯一入口
- 启动前台主控程序

### 3.2.2 RemoteSshApp.ps1

作用：

- 显示中文流程状态
- 显示每个步骤是否完成
- 显示详细日志
- 展示最终成功或失败结果

它是用户始终能看到的主窗口。

### 3.2.3 RemoteSshWorker.ps1

作用：

- 负责真正执行需要管理员权限的任务
- 检查 OpenSSH 是否存在
- 在缺失时启动独立安装器
- 启动 `sshd`
- 配置防火墙
- 写入授权公钥
- 调用注册接口
- 启动反向 SSH 隧道

它是提权后的后台执行器。

### 3.2.4 InstallOpenSsh.ps1

作用：

- 只负责安装 `OpenSSH Server`
- 输出安装日志
- 支持被后台隐藏调用
- 也支持单独弹出窗口执行

它的日志会被主控窗口读取并显示。

## 3.3 Windows 流程

用户双击后，流程如下：

1. 启动前台主控窗口
2. 主控窗口拉起提权后的后台执行器
3. 后台执行器检查 `OpenSSH Server`
4. 如果已安装，则直接进入后续步骤
5. 如果未安装，则启动独立安装器
6. 安装器后台执行或单独开窗执行
7. 主控窗口持续显示安装日志
8. 安装完成后继续启动 `sshd`
9. 配置防火墙与本机授权
10. 向中转服务器注册
11. 建立反向 SSH 隧道
12. 输出最终 SSH 命令

## 3.4 安装模式设计

通过 `windows/config.ini` 中的 `INSTALL_OPENSSH_MODE` 控制：

- `hidden`
  - 独立安装器在后台执行
  - 主控窗口显示日志

- `cmd`
  - 独立安装器在单独的 `cmd` 窗口中执行
  - 主控窗口仍然显示日志

- `window`
  - 兼容旧配置
  - 内部会按 `cmd` 模式处理

这样既满足“后台安装”的需求，也满足“另外弹出一个窗口安装”的需求。

## 3.5 状态与日志设计

### 3.5.1 status.json

用于描述：

- 当前步骤
- 每个步骤状态
- 当前详细日志文件位置
- 整体成功或失败

### 3.5.2 worker.log

用于记录后台执行器自己的流程日志。

### 3.5.3 install-openssh.log

当 OpenSSH 进入安装阶段时，独立安装器会写入这个文件。

主控窗口在安装期间会切换到这个日志文件，把它展示给用户。

## 4. 安全边界

客户端包内不包含：

- 中转服务器 root 密码
- 中转服务器可交互管理权限

客户端包只包含：

- 中转服务器地址
- 注册接口地址
- 注册码
- 操作员公钥

因此，客户端即使被转发，也不会直接泄露服务器 root 权限。

## 5. 已知边界

1. Windows 当前采用系统 `OpenSSH Server` 路线，不是内置第三方 SSH 服务。
2. macOS 当前依赖系统自带 Remote Login。
3. 目前已经实现一键接入、自动安装、日志展示、隧道建立。
4. 断线重连守护和长期运维能力还可以后续继续增强。
