# Windows 使用文档

## 1. 适用对象

本文档用于指导目标用户在 Windows 电脑上启动一键接入工具。

目标用户只需要：

- 收到一份启动包
- 双击启动
- 在提权提示时点允许
- 把最终显示出来的 SSH 命令发给操作员

## 2. 启动前提

建议系统：

- Windows 10
- Windows 11
- 具备管理员授权能力

首次运行时，程序可能会：

- 安装 OpenSSH Server
- 启动 `sshd`
- 配置防火墙规则

因此会弹出管理员权限确认。

## 3. 启动方式

双击：

```text
start-remote-ssh.bat
```

这会启动一个 PowerShell 控制窗口。

## 4. 程序会做什么

Windows 启动器会按顺序完成以下步骤：

1. 检查管理员权限
2. 检查 OpenSSH Server 是否已安装
3. 如未安装，则自动安装
4. 启动 `sshd` 服务并设置为自动启动
5. 放行本机防火墙 22 端口
6. 检查 `127.0.0.1:22` 是否可访问
7. 生成设备密钥
8. 写入操作员公钥
9. 向中转服务器注册
10. 启动反向 SSH 隧道
11. 输出最终连接命令

## 5. 成功后的表现

如果执行成功，窗口里会看到类似内容：

```text
Connection ready.
Send this command to the admin:
ssh -p 24001 用户名@106.13.171.166
```

把这条命令原样发给操作员即可。

## 6. 配置文件说明

配置文件：

```text
config.ini
```

主要字段：

- `RELAY_HOST`：中转服务器地址
- `RELAY_SSH_PORT`：中转服务器 SSH 端口
- `ENROLL_API`：注册接口地址
- `ENROLL_CODE`：注册码
- `ADMIN_PUBLIC_KEY`：操作员公钥
- `DRY_RUN`：是否使用演练模式

## 7. 演练模式

如果需要先本地演示流程，不改动系统，可设置：

```ini
DRY_RUN=true
```

演练模式下会模拟：

- 安装 SSH 服务
- 配置防火墙
- 注册成功
- 隧道建立成功

但不会真正改系统配置。

## 8. 常见问题

### 8.1 提示权限不足

说明当前没有以管理员身份完成提权。请重新双击启动，并在 UAC 弹窗中选择允许。

### 8.2 注册成功但操作员连不上

常见原因：

1. 反向隧道没有成功建立
2. 中转服务器安全组没有放行映射端口
3. 本机 `sshd` 虽启动了，但密钥权限或登录用户名不对

### 8.3 本机可以打开窗口，但很快失败

建议查看运行日志：

```text
%LOCALAPPDATA%\RemoteSshRelay\runtime\worker.log
```

以及状态文件：

```text
%LOCALAPPDATA%\RemoteSshRelay\runtime\status.json
```

## 9. 注意事项

1. 关闭控制窗口不会自动删除已经启动的系统 SSH 服务。
2. 如果你不再希望被远程连接，建议停止隧道进程并清理授权公钥。
3. 该工具更适合临时远程接入，不建议在不知情的情况下长期常驻。
