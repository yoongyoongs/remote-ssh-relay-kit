# 工作状态文档

更新时间：`2026-06-13`

## 1. 当前结论

项目主体已经可用，Windows 端重设计目标已经落地：

1. 缺少 OpenSSH 时可自动安装
2. 安装过程可后台执行，或单独弹出 `cmd` 窗口执行
3. 主程序会持续显示中文步骤、当前状态和安装日志

但整条“公网真实链路”还差最后一层外网放通：

1. 服务器内网 `127.0.0.1:8787` 正常
2. 服务器公网 `106.13.171.166:8787` 仍然超时
3. 所以真实的 Windows 注册动作还不能直接走公网完成

### 3.5 当前失败记录

在 2026-06-13 的最新验证里：

1. Windows 本机执行 `curl --noproxy * http://106.13.171.166:8787/health` 超时
2. Windows 本机执行 `Test-NetConnection 106.13.171.166 -Port 8787` 提示连接失败
3. Linux 服务器自己执行 `curl http://106.13.171.166:8787/health` 也超时

这个现象说明问题更偏向：

1. 云安全组未放开 `8787/TCP`
2. 云防火墙有额外限制
3. 不是中继 API 进程没启动
4. 在你关闭 TUN 后，仍然复现，所以已经不能再优先怀疑 Clash TUN
