### 3.11 2026-06-17 Windows 7 启动卡死排查与诊断包增强 (已落地)

在 Windows 7 真机验证中，用户反馈主窗口长期停留在：

```text
[进行中] 正在等待后台任务启动...
详细日志
  正在等待日志输出...
```

该现象说明前台 `RemoteSshApp.ps1` 已启动，但后台 `RemoteSshWorker.ps1` 未能及时写出 `status.json` 或 `worker.log`。本阶段完成以下修复：

1. **PowerShell 2.0 / .NET 3.5 残留兼容修复**：
   - 移除 helper 中对 `[System.Management.Automation.PSCustomObject]` / `[PSCustomObject]` 的依赖，统一使用 `New-Object PSObject` 与通用属性枚举。
   - 将 `TcpClient.ConnectAsync()` 替换为 .NET 3.5 可用的 `BeginConnect()` / `EndConnect()`。
   - 同步更新 `scripts/apply-compat-fixes.js`，避免后续重新生成兼容代码时把旧问题带回来。
2. **后台启动失败兜底**：
   - 前台主程序在父进程已是管理员时记录 worker 进程句柄。
   - 若 worker 启动后立即退出，会写入失败结果，不再无限等待。
   - 若 45 秒内仍未看到 `status.json`，会返回超时失败，并提示可能是 UAC 未确认、PowerShell 兼容错误或安全软件拦截。
3. **面向非技术用户的自动诊断包**：
   - 失败时自动在桌面生成 `RemoteSshRelay-Diagnostics-会话ID.zip`。
   - 自动打开资源管理器并选中文件，同时把路径复制到剪贴板。
   - 用户只需把该 zip 文件发给管理员，不需要自行进入 `%LOCALAPPDATA%` 查日志。
   - 若 zip 创建失败，自动退化为桌面同名诊断文件夹。
4. **Windows 7 OpenSSH 边界提示**：
   - `InstallOpenSsh.ps1` 检测不到 `Get-WindowsCapability` 时，会明确提示 Windows 7 不能通过 Windows 可选功能自动安装 OpenSSH Server。
   - 若 Win7 机器没有 `sshd` 服务，需要先手工安装 Win32-OpenSSH，或改用已预装 SSH 服务的环境。
5. **默认配置回到方案B**：
   - `windows/config.ini` 中 `ENROLL_API` / `BOOTSTRAP_API` 已改为 `http://106.13.171.166:8787/...`。
   - `RELAY_HOST` 继续保持 `yoong-relay.ddnsgeek.com`。
