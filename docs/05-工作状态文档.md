## 4. 当前未完成或建议继续加强的部分

1. 还没有实现完整的断线自动重连守护。
2. 还没有做 HTTPS 和更强的注册认证机制。
3. macOS 脚本目前更偏轻量方案，后续可以继续增强日志与异常提示。
4. 真机联调仍建议至少对一台 Windows、一台 macOS 各做一次完整穿透测试。

## 5. 风险提示

1. 注册码属于轻量接入控制，不应长期固定。
2. 中转服务器需要正确开放安全组和本机防火墙端口。
3. 若操作员私钥管理不当，目标机器访问权限也会跟着暴露。
4. 该方案本质上是自建中转，不等同于成熟商用远控平台。

## 6. 交付建议

建议交付时按以下方式使用：

1. 先部署中转服务器
2. 生成操作员 SSH 密钥对
3. 用 `prepare-launchers.ps1` 写入注册码和公钥
4. 用 `build-release.ps1` 打出压缩包
5. 把 Windows 或 macOS 启动包分别发给目标用户
6. 由目标用户把最终 SSH 命令回传给操作员
