### 3.7 当前失败记录

在 2026-06-15 的最新验证里：

1. Windows 本机执行 `curl --noproxy * http://106.13.171.166:8787/health` 超时
2. Windows 本机执行 `ssh -p 24006 Admin@106.13.171.166` 超时
3. 服务器自己已经监听 `24006`，所以更像云安全组或云防火墙仍未放开 `24000-24999/TCP`
4. 当前本机是管理员账号，但本轮验证使用的是 `DRY_RUN_USE_LIVE_RELAY=true` 的非提权演练模式，因此不会去写 `C:\ProgramData\ssh\administrators_authorized_keys`
5. Windows 本机 `sshd_config` 明确包含：
   - `Match Group administrators`
   - `AuthorizedKeysFile __PROGRAMDATA__/ssh/administrators_authorized_keys`
6. 这意味着“管理员私钥真正登录 Windows”这一层仍需在提权的正式模式下再验一次

这个现象说明问题更偏向：

1. 云安全组未放开 `8787/TCP`
2. 云安全组未放开 `24000-24999/TCP`
3. 云防火墙有额外限制
4. 正式提权模式下还需要把管理员公钥写入 `administrators_authorized_keys`
5. 由于当前 Codex 会话本身不是提权会话，且系统开启了 UAC（`EnableLUA=1`，`ConsentPromptBehaviorAdmin=5`），最后一步正式管理员授权写入需要桌面端人工同意 UAC
