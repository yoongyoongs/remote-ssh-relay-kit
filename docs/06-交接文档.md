## 11. 方案B（国内免备案域名阻断规避机制）与域名回切指南

### 11.1 方案B 规避机制说明
由于本项目的公网中转服务器部署在国内百度云，且免费动态域名（如 `ddnsgeek.com`）没有国内 ICP 备案，直接使用域名发起 HTTP API 请求（如 `/api/bootstrap` 和 `/api/enroll`）会被百度云的网关强行拦截阻断，返回备案提示页面。

为了解决此阻塞，我们实施了**方案B**：
- **API 请求直连 IP**：将客户端的 API 地址（`ENROLL_API` 和 `BOOTSTRAP_API`）硬编码绑定为服务器的真实公网 IP 地址（`http://106.13.171.166:8787`）。由于 IP 地址请求不带 Host，云厂商网关不会进行 Host 域名拦截，从而保证 API 正常握手。
- **SSH 隧道连接走域名**：反向 SSH 隧道所使用的 `RELAY_HOST` 和返回给操作员的远程连接命令，依旧保留使用域名（如 `yoong-relay.ddnsgeek.com`）。因为 SSH 连接属于非 HTTP 协议流量（通常是 TCP 端口 22 或 24000-24999），可以直接穿透网关，不受备案网关的域名拦截限制。

### 11.2 后续域名通过备案审批后的“域名回切”操作指南
一旦用户在百度云申请的新域名（或当前域名）通过了国内实名认证与 ICP 备案审批，我们可以非常方便地将客户端 API 接口地址**重新切回域名**。

操作员在本地机器上的回切操作步骤如下：

1. 打开操作员主机的 PowerShell 终端。
2. 运行打包脚本，**不要指定 `-ApiHost` 参数**（此时 `-ApiHost` 默认留空，脚本将自动把 API 地址配置为与中继域名一致）：
   ```powershell
   powershell -File scripts/prepare-launchers.ps1 `
     -BootstrapToken RLY-TOKEN-2026-AUTOMATION `
     -RelayHost 您的新域名.com
   ```
3. 重新压缩打包客户端：
   ```powershell
   powershell -Command "Compress-Archive -Path release/launcher-kit/windows/* -DestinationPath release/launcher-kit-windows-v2.zip -Force"
   ```
4. 将新生成的 `launcher-kit-windows-v2.zip` 重新分发给被协助用户运行。此时整个系统将完全实现全域名（API 与 SSH 连接均为域名）安全访问，完美完成过渡。

## 12. 2026-06-17 Windows 7 真机验证问题与当前处理结论

### 12.1 现场现象

Windows 7 真机运行启动器时，主窗口卡在：

```text
[进行中] 正在等待后台任务启动...
详细日志
  正在等待日志输出...
```

这表示前台主控窗口已启动，但后台 worker 没有成功写出 `status.json` 或 `worker.log`。原版本缺少启动失败兜底，所以用户会看到窗口一直等待。

### 12.2 已完成修复

1. **PowerShell 2.0 兼容残留清理**：
   - 移除了脚本中残留的 `[PSCustomObject]` / `[System.Management.Automation.PSCustomObject]` 依赖。
   - 将 `TcpClient.ConnectAsync()` 改为 .NET 3.5 可用的 `BeginConnect()` / `EndConnect()`。
2. **后台启动失败兜底**：
   - worker 启动即退出时会生成失败结果。
   - 等待超过 45 秒仍无状态文件时会生成失败结果。
3. **自动诊断包**：
   - 失败时在桌面生成 `RemoteSshRelay-Diagnostics-会话ID.zip`。
   - 自动打开资源管理器选中该文件，并把路径复制到剪贴板。
   - 被协助用户只需要把这个文件发给管理员。
4. **方案B 默认配置落地**：
   - Windows 默认 `ENROLL_API` 和 `BOOTSTRAP_API` 已改为公网 IP `106.13.171.166`。
   - SSH 中继地址仍使用 `yoong-relay.ddnsgeek.com`。
5. **OpenSSH 安装边界提示**：
   - Windows 7 缺少 Windows 10/11 的 OpenSSH 可选功能安装机制。
   - 如果目标机没有 `sshd` 服务，当前版本会明确提示需要先安装 Win32-OpenSSH。

### 12.3 当前结论

本轮是 **Windows 7 启动诊断修复版**，不是 Windows 7 完整一键安装闭环版。

如果 Win7 已经存在可用的 `sshd` 服务，修复后的启动器有机会继续完成后续 Bootstrap、注册和反向隧道流程。

如果 Win7 没有 `sshd` 服务，后续需要新增一条 Win32-OpenSSH 路线：

1. 将 Win32-OpenSSH 安装包纳入分发包，或提供可下载的离线安装包。
2. 新增 Win7 专用安装脚本，完成解压、安装服务、设置权限、启动 `sshd`。
3. 完成 `C:\ProgramData\ssh\administrators_authorized_keys` 与普通用户 `authorized_keys` 的兼容写入验证。
4. 再进行 Win7 真机完整链路验收。
